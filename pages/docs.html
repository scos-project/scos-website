Project Overview

SCOS is a Windows-based system designed to turn a PC into a living-room gaming console. It boots straight into Steam Big Picture Mode and hides desktop clutter so you can sit back and play with a controller.

  • Native Windows compatibility — full support for Windows games & launchers (no Proton layer).
  • Console-like simplicity — direct boot to Steam Big Picture.
  • Controller-first — daily navigation without keyboard/mouse.
In short: SteamOS is for the Steam Deck. SCOS is for your living-room PC console.

Installation

  • Use the provided SCOS ISO to install on your PC.
  • Install SCOS before connecting controllers.
  • After installation completes and the system is running, connect controllers via USB.

Minimum & Recommended Requirements

Minimum

  • CPU: Dual-Core (Intel i5 4th gen or higher recommended)
  • RAM: 8 GB
  • GPU: DirectX 12 compatible (GTX 900 / RX 400 or newer)
  • Storage: 60 GB free space
  • Ethernet: Required for first boot (Wi-Fi not directly supported)
  • USB Hub: Recommended for multiple controllers

Recommended (Modern Games)

  • CPU: Quad-Core (Intel i7 / Ryzen 5 or better)
  • RAM: 16 GB
  • GPU: NVIDIA RTX 2000 / AMD RX 6000 or better
  • Storage: 1 TB SSD
  • Network: Gigabit Ethernet or TP-Link repeater via Ethernet

Networking

  • First Boot: Ethernet required.
  • Wi-Fi Alternative: Use a TP-Link Wi-Fi repeater (or similar): configure it to your Wi-Fi, then link it by Ethernet to SCOS.
  • Direct Wi-Fi configuration in SCOS is not available.

Controllers

  • USB only (wired) in this beta.
  • Use a USB hub to connect multiple controllers.
  • Bluetooth support is planned for a future version.

Pre-Installed Software (Beta)

  • Steam — launches in Big Picture Mode as the main UI
  • EA App — required for some EA titles
  • Unified Remote (Server) — control keyboard/mouse from your phone to handle external sign-ins (EA, Rockstar, etc.)

Known Bug: In this beta, the Unified Remote installer may be detected but not actually installed automatically.

  • Manual installation is not possible after SCOS setup due to the console-style lockdown.
  • This will be fixed in a future release.

More apps (Epic, Rockstar, Ubisoft, etc.) are planned for future releases.

Console-Mode Design

SCOS is designed to behave like a console, not a desktop PC. Background behavior is optimized for stability and simplicity: power on, play, relax.

  • Boots directly into Steam Big Picture and stays there for daily use.
  • No desktop/taskbar workflow; controller-first navigation.

Settings Access

The only Windows setting intended for access is Microphones & Speakers:

  1. Press the controller Home button to open the Steam Menu.
  2. Go to Settings → In-Game Voice.
  3. Under Microphone, choose Select Device to open the Sound settings pop-up.

Controller navigation in Sound popup: use your controller’s mouse-emulation (e.g., hold PS button + right stick), RB = left click, LB = right click.

Steam Behavior

  • Exiting Big Picture Mode opens the Steam Desktop Mode window.
  • Exiting Steam entirely can leave a black screen.
  • Restarting always returns to Big Picture at boot (SCOS sets it as the shell).

Quick Start Guide (Beta)

  1. Create a bootable USB (≥ 16 GB) with:
    • Rufus (Windows)
    • BalenaEtcher (Windows/macOS/Linux)
    Select the SCOS ISO.
  2. Boot from the USB and install SCOS.
  3. Plug an Ethernet cable (required for setup).
  4. Wait while GPU drivers install via Windows Update (screen may flicker).
  5. Restart after drivers finish (recommended).
  6. SCOS boots into Steam Big Picture. Sign in to Steam.
  7. EA/Rockstar sign-ins normally use Unified Remote — not functional in this beta (see known bug).
  8. Connect controllers via a USB hub and start gaming.
  9. To adjust audio devices: Steam Menu → Settings → In-Game Voice → Microphone → Select Device.

Privacy & Data

SCOS keeps things simple: no analytics by default, and the OS aims to minimize/disable telemetry where safe.

  • Contact form data: Our website’s contact form is embedded from an external provider (e.g., Google Forms, Typeform, Tally). When you submit, your data is processed and stored by that provider—not by SCOS.
  • What’s collected: Whatever you enter in the form (e.g., name, email, message) plus any metadata the provider gathers (timestamps, IP per provider policy). Avoid sharing sensitive information.
  • Retention & deletion: Managed by the form provider. Refer to their privacy policy for details. If you need data removed, contact us and we’ll forward the request to the provider.
  • Website analytics: None enabled by default. If we add privacy-friendly analytics later (e.g., Plausible/Umami), we’ll document it here.
  • Telemetry in SCOS (OS build): We aim to keep the environment bloat-free and reduce telemetry where it’s safe to do so for a console-like experience.

Questions about data handling? Reach us via the Contact page.
